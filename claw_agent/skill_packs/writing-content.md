# Writing & Content Skill Pack

## Writing Principles
- **Clarity over cleverness**: If a sentence requires re-reading, rewrite it
- **Active voice**: "The system processes requests" not "Requests are processed by the system"
- **Concrete over abstract**: "Response time dropped from 2s to 200ms" not "Performance improved significantly"
- **Cut ruthlessly**: Every word earns its place. Remove "very", "really", "basically", "actually"
- **One idea per paragraph**: Topic sentence → support → transition to next idea

## Content Types
- **Technical Blog Posts**: Problem → Context → Solution → Code → Results → Takeaways
- **Documentation**: Task-oriented. What the user wants to DO, not what the system IS
- **Tutorials**: Step-by-step with complete code. Every step must produce a verifiable result
- **Case Studies**: Situation → Challenge → Solution → Results (with numbers)
- **API Docs**: Endpoint, method, parameters, request/response examples, error codes

## Structure
- **Headlines**: Specific, benefit-oriented. "How to Deploy to Production in 5 Minutes"
- **Opening**: Hook in first 2 sentences. State the problem, promise the solution
- **Body**: Subheads every 200-300 words. Short paragraphs (3-5 sentences max)
- **Conclusion**: Summarize key takeaways. Clear next action. No new information
- **Formatting**: Lists for >3 items. Code blocks for code. Bold for emphasis, sparingly

## Voice & Tone
- **Match the audience**: Developer docs ≠ marketing copy ≠ support articles
- **Avoid AI-isms**: No "dive into", "leverage", "in today's fast-paced world", "it's worth noting"
- **Be direct**: Say "Use X" not "You might want to consider using X"
- **Conversational but precise**: Write like talking to a smart colleague, not a thesis committee

## Editing Checklist
1. Read aloud — cut wherever you stumble
2. Remove weasel words: "some", "various", "certain", "often"
3. Replace jargon with plain language (or define it on first use)
4. Check every heading delivers what it promises
5. Verify all code examples compile and run
