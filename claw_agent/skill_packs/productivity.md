# Productivity Skill Pack

## Project Planning
- **Blueprint Method**: One-line objective → step-by-step plan with context briefs per step
- **Vertical Slices**: Deliver thin, end-to-end increments. Not horizontal layers
- **Estimation**: Break into tasks < 4 hours. If bigger, decompose further. Use T-shirt sizing for roadmap
- **Dependencies**: Map them explicitly. Critical path determines timeline. Parallelize non-critical
- **Risk Register**: Identify risks early. Mitigation plan for top 3. Accept the rest

## Task Management
- **Todo States**: Not started → In progress (1 at a time) → Done. WIP limits prevent context switching
- **Priority**: Urgent+Important (do now), Important (schedule), Urgent (delegate), Neither (delete)
- **Timeboxing**: Set a fixed time budget. Ship what you have when time's up
- **Definition of Done**: Tests pass, code reviewed, documented, deployed. Not "I think it works"

## Code Review
- **Author prep**: Small PRs (<400 lines). Clear description. Link to issue/spec
- **Reviewer focus**: Correctness → Security → Performance → Style. In that priority order
- **Feedback style**: Ask questions before making demands. "What happens if X?" > "This is wrong"
- **Turnaround**: < 24 hours for review. Block nothing else while waiting

## Documentation
- **ADRs**: Architecture Decision Records for non-obvious choices. Context → Decision → Consequences
- **READMEs**: What is this? How do I run it? How do I contribute? Where can I get help?
- **Runbooks**: Step-by-step for operational procedures. Tested by someone who didn't write them
- **Changelogs**: User-facing. What changed, why it matters, what to do

## Knowledge Management
- **Session notes**: Capture decisions, blockers, and context at the end of each work session
- **Memory system**: User prefs (persistent), session notes (temporary), repo knowledge (shared)
- **Onboarding docs**: Architecture map, key entry points, conventions, build commands
- **Skill extraction**: When you solve something hard, extract the pattern into a reusable skill
